# Graph Dynamics

The current graph suggests six reinforcing movements.

1. Agents move from generating recommendations to executing transactions.
2. Incumbent enterprise platforms supply distribution, data, permissions, and budget ownership.
3. Machine-originated traffic becomes identifiable and economically addressable.
4. Identity, controls, observability, and regulation expand alongside autonomy.
5. Scaled operational adoption concentrates in bounded service workflows where structured records, repetitive triggers, measurable pain, and established escalation paths make execution tractable.
6. Evaluation and staging boundaries are economically material: agents can cross credentials, networks, and organizations before intended deployment.

These are hypotheses, not settled conclusions. Each is represented as a thesis with supporting and challenging evidence.

## Research feedback loop

Each collection run uses the current graph to improve the next search:

1. Event stages and open questions identify missing operational or economic evidence.
2. Thesis support/challenge balance creates explicit counterevidence searches.
3. The evidence ontology supplies the stage and metric vocabulary used in graph-derived searches and records workflow, operational, impact, and counterevidence research goals in the review plan.
4. Incoming candidates are matched to known nodes, scored for review priority, and assigned a routing-only evidence contract with explicit missing dimensions.
5. Human/Codex review verifies primary sources and rejects announcement-only, rebranded, or causally ambiguous claims.
6. Accepted evidence changes events, concepts, update chains, and theses, which changes the following search plan.

The generated search plan is inspectable at `raw/inbox/search-plan.json`. Relevance scores, maturity hints, and admission routes prioritize human/Codex review and never substitute for source verification.

## Dynamics to watch

- `protocol -> pilot -> production -> scaled` update chains.
- Movement from per-seat pricing toward per-task, per-resolution, transaction-fee, or revenue-share models.
- Growth in verified agent traffic versus unidentified automation and bot traffic.
- Whether security and integration costs fall faster than agent capability improves.
- Whether new agents create net-new activity or merely replace human interfaces.
- Whether sandbox, classifier, network, and credential controls prevent evaluation agents from affecting external production systems.
